Learning from data

As weve learned, in machine learning, we typically create models that predict or classify input values. We can think of a model as a function or an equation that describes the relationship between the input and output.

For example: $$ y = 2x + 3 $$ This is a simple equation where y is the output and x is the input.

More complex models might have multiple inputs: $$ y = 2x_1 + 3x_2 + 4 $$

The general form of the fucntion is: $$ y = mx + c $$

You are familar with this form of the function from your algebra classes. Here,

$y$ is the output, $x$ is the input, $m$ is the weight, $c$ is the intercept.

In machine learning we write typically write this function slightly differently: $$ y = wx + b $$

or in case of multiple inputs: $$ y = w_1x_1 + w_2x_2 + ... + w_nx_n + b $$

Here, $w$ is the weight, $x$ is the input, $b$ is the intercept.

Our goal in machine learning is to learn the weights (like 2 and 3) and the intercept (like 4) from data, allowing us to make predictions.

The Three Key Parts of a Machine Learning Algorithm

Before diving deeper into machine learning algorithms like Linear Regression and Logistic Regression, its important to understand that all machine learning algorithms share three key components:

  1. The Function (or Model): This is the equation or formula as depicted above, that represents the relationship between the input data and the output predictions. The function helps us make predictions based on input data.
  2. The Loss Function: This tells us how well the model is doing at making predictions. The loss function measures the difference between the models predictions and the actual data. The goal is to minimize this loss as much as possible.
  3. The Optimization Function: This is the process that helps us improve the model by adjusting its parameters to minimize the loss. The optimization function systematically updates the model to make better predictions.

A Simple Example to Illustrate

Imagine youre hiking down a hill, and your goal is to reach the lowest point before it gets dark. Lets break down how this situation relates to the three key parts of a machine learning algorithm:

  1. The Function (or Model): Think of this as a map that describes the hills shape. The map tells you where you are on the hill and the slope at each point. For example, you could use the function below. Here, x is your position on the hill, and y is the height. The slope m and the intercept c describe the shape of the hill. $$ y = mx + c $$

  2. The Loss Function: The loss function tells you how far you are from the bottom of the hill (the optimal point). If youre far from the bottom, your loss is high. If youre close to the bottom, the loss is low. In this example, the loss is the height of the hill: the higher up you are, the greater the loss.

  3. The Optimization Function: The optimization function is like your strategy for getting down the hill. You might use a strategy like follow the steepest path downhill. This is similar to Gradient Descent in machine learning, where the model looks for the steepest direction to reduce the loss. With each step, you adjust your position (the models parameters) to get closer to the lowest point (minimize the loss).

Applying This to Machine Learning

In machine learning, we follow a similar process:

  • The Function (or Model): This could be a linear function like y = mx + c in Linear Regression. It predicts the output y based on the input x .
  • The Loss Function: This could be Mean Squared Error (MSE) in Linear Regression, which tells us how far the predicted values \hat{y} are from the actual values y . The higher the MSE, the worse the model is doing.
  • The Optimization Function: This could be Gradient Descent, which helps adjust the parameters m and c to minimize the loss function. Each time we update the parameters, we get a little closer to a better model that predicts the data more accurately.

Loss Functions

Whenever we build a model, we need to evaluate how well it works. This is done using loss functions. A loss function tells us how far off our model's predictions are from the actual results. The lower the loss, the better the model is.

The most common loss function is the Mean Squared Error (MSE).

Mean Squared Error (MSE)

For regression problems, where we predict continuous values (like house prices or test scores), we use Mean Squared Error (MSE). MSE measures the average squared difference between the predicted values ($\hat{y}$) and the actual values ($y$):

$$ \text{MSE} = \frac{1}{n} \sum_{i=1}^{n} (y_i - \hat{y}_i)^2 $$

Here's an example using actual values to make this more concrete:

  • Suppose we have data points: $x = [1, 2, 3]$ and corresponding actual values $y = [2, 4, 6]$.
  • Let's assume our model's predictions are $\hat{y} = [1.8, 4.2, 5.9]$.

To compute the MSE:

$$ \text{MSE} = \frac{1}{3} \left[(2 - 1.8)^2 + (4 - 4.2)^2 + (6 - 5.9)^2 \right] $$

Breaking it down:

  • $(2 - 1.8)^2 = 0.04$
  • $(4 - 4.2)^2 = 0.04$
  • $(6 - 5.9)^2 = 0.01$

Summing them and averaging: $$ \text{MSE} = \frac{1}{3} (0.04 + 0.04 + 0.01) = 0.03 $$

This tells us that, on average, our model's predictions are off by about 0.03 units from the actual values.

Binary Cross-Entropy

For classification problems, where the goal is to predict categories (like pass or fail), we use a different loss function called Binary Cross-Entropy. This measures how well the model is predicting probabilities for binary outcomes (0 or 1).

The formula is:

$$ L = - \frac{1}{n} \sum_{i=1}^{n} \left[ y_i \log(\hat{y}_i) + (1 - y_i) \log(1 - \hat{y}_i) \right] $$

Heres an example using actual values:

  • Suppose we have binary outcomes y = [0, 1, 1] and our models predicted probabilities are \hat{y} = [0.2, 0.8, 0.9] .

To compute the Binary Cross-Entropy loss:

$$ L = - \frac{1}{3} \left[ 0 \cdot \log(0.2) + 1 \cdot \log(0.8) + 1 \cdot \log(0.9) \right] $$

Breaking it down:

  • For the first data point: 0 \cdot \log(0.2) = 0
  • For the second data point: \log(0.8) = -0.223
  • For the third data point: \log(0.9) = -0.105

Summing and averaging:

$$ L = - \frac{1}{3} \left[ 0 + (-0.223) + (-0.105) \right] = 0.109 $$

This value represents the average error in predicting the binary class probabilities. The closer the loss is to 0, the better the model is performing.

Linear Regression

In Linear Regression, we try to fit a straight line to the data points, and we try to minimize the Mean Squared Error (MSE) between the predicted values and actual values.

The equation of the line is: $$ y = mx + c $$

Where:

  • $y$ is the predicted output,
  • $x$ is the input,
  • $m$ is the slope,
  • $c$ is the intercept.

Ordinary Least Squares (OLS) Method

The Ordinary Least Squares (OLS) method is a way to find the best-fit line by minimizing the MSE directly. It calculates the slope $m$ and intercept $c$ by solving for the values that make the squared differences between the actual values and the predicted values as small as possible.

In OLS, we derive formulas for $m$ and $c$:

$$ m = \frac{\sum{(x_i - \bar{x})(y_i - \bar{y})}}{\sum{(x_i - \bar{x})^2}} $$ $$ c = \bar{y} - m \cdot \bar{x} $$

Where $\bar{x}$ and $\bar{y}$ are the means of the $x$ and $y$ values.

$$ \hat{y} = \frac{1}{1 + e^{-z}} $$

Where $z = mx + c$.

We typically solve for $m$ and $c$ using Maximum Likelihood Estimation (MLE), but let's use a simplified example where we already know the values for $m$ and $c$.

Introducing Gradient Descent

Now that we've explored Linear and Logistic Regression using traditional methods (like Ordinary Least Squares (OLS) for Linear Regression), let's introduce a more general method to optimize our models: Gradient Descent.

What is Gradient Descent?

Gradient Descent is an optimization algorithm used to minimize a loss function by iteratively adjusting the model's parameters (like $m$ and $c$ in Linear Regression). It's like a hiker trying to find the lowest point on a hill. At each step, the hiker looks at the steepness of the slope (the gradient) and takes a step downhill. Over time, the hiker gets closer and closer to the bottom of the hill.

For machine learning models, this means adjusting the parameters (weights) in small steps, moving in the direction that reduces the error (loss).

The Update Rule

In Gradient Descent, we update the parameters using the following rule:

$$ \theta = \theta - \eta \cdot \frac{\partial L}{\partial \theta} $$

Where:

  • $\theta$ represents the model parameters (like $m$ and $c$),
  • $\eta$ is the learning rate (how big each step is),
  • $\frac{\partial L}{\partial \theta}$ is the gradient of the loss function with respect to the parameters.

The learning rate $\eta$ controls how large the steps are during each iteration. If the steps are too small, it will take a long time to converge; if they are too large, the optimization might overshoot and not converge at all.

# Define the loss function (Mean Squared Error)
def mean_squared_error(y_true, y_pred):
    return np.mean((y_true - y_pred) ** 2)

# Define the gradient descent algorithm for Linear Regression
def gradient_descent(x, y, learning_rate=0.01, epochs=1000):
    m, c = 0, 0  # Initialize slope and intercept
    n = len(y)    # Number of data points
    
    # Perform Gradient Descent
    for _ in range(epochs):
        y_pred = m * x + c  # Predicted values based on current m and c
        # Calculate the gradients
        dm = (-2/n) * np.sum(x * (y - y_pred))  # Gradient of m
        dc = (-2/n) * np.sum(y - y_pred)        # Gradient of c
        # Update m and c
        m -= learning_rate * dm
        c -= learning_rate * dc
        
        # Optional: Print loss during training for debugging
        if _ % 100 == 0:
            mse = mean_squared_error(y, y_pred)
            print(f'Epoch {_}: m = {m:.4f}, c = {c:.4f}, MSE = {mse:.4f}')
    
    return m, c

Conclusion: Gradient Descent in Practice

In this section, we've extended the Linear Regression example to use Gradient Descent instead of Ordinary Least Squares. Gradient Descent is a more flexible method that can be used to optimize complex models where direct solutions like OLS are not feasible.

Why Gradient Descent?

  • General-purpose optimization: Gradient Descent works for many types of models, including those that cannot be solved analytically.
  • Scalability: In complex models with many parameters, Gradient Descent is often the only feasible way to find the optimal solution.

This is especially true when we extend this approach to Logistic Regression and more advanced machine learning models, where direct methods like OLS are not applicable.
